# 인사이트: GEO/AEO — AI 에이전트가 찾는 사이트

## 관찰 (2026-06-11)
- 네이버 지도에서 카페 검색해도 "홈페이지" 클릭은 거의 없음 — 블로그/SNS가 더 효과적
- 대기업 카페 프랜차이즈도 홈페이지 퀄리티가 의외로 낮음
- 모바일 트래픽이 압도적으로 큼

## 다음 단계 가설
"예쁜 홈페이지"의 전통적 효용은 약해지고 있고, 다음은:
- **GEO/AEO (Generative/Answer Engine Optimization)**: AI 에이전트가 가게 정보를 정확히 읽고 추천/예약까지 할 수 있는 사이트
- 사람도 찾아오고 AI도 추천하는 사이트 = 11월 강의 차별화 포인트가 될 수 있음

## 지금 적용 가능한 것 (우선순위 순)
1. **구조화 데이터 (Schema.org/JSON-LD)** — 가게명, 주소, 영업시간, 메뉴, 가격을 `<script type="application/ld+json">`으로 명시
2. **명확한 텍스트 콘텐츠** — 이미지 대신 텍스트로 메뉴/가격/특징 명시 (AI는 이미지 속 텍스트를 잘 못 읽음)
3. **llms.txt** — AI 크롤러용 사이트 요약 파일 (아직 초기 표준)

## 미래 비전 (11월 강의에서 "보여주는" 수준으로만)
- 사이트 자체가 작은 MCP 서버 역할 → AI 에이전트가 "예약 가능 여부", "메뉴 조회"를 직접 호출
- 지금 데모에 구현하긴 이르지만, 강의에서 "이런 미래도 있다"고 비전 챕터로 소개 가능

## 오늘 데모에 반영할 것
- JSON-LD로 가게 정보(LocalBusiness) 구조화 데이터 추가
- 메뉴/가격을 이미지가 아닌 텍스트로 명시
- "오시는 길"은 사이트 내 지도 embed 대신 → 네이버 지도 길찾기 버튼 + 주소 텍스트
  (이유: 지도 보고 찾아온 사람에겐 사이트 내 지도가 불필요, SNS 유입자에겐 "연결 통로"면 충분, 디자인도 깔끔해짐)
